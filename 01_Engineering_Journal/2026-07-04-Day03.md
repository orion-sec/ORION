# ORION Engineering Journal

**Date:** 04 July 2026
**Day:** 3
**Project:** ORION Security Platform
**Company:** ORION SECURITY LABS LTD


What I Learned Today

Today I gained a much clearer understanding of the engineering tools that will be used to build ORION.

I learned that Git is much more than a backup tool. It records every change made to the source code, allowing developers to safely experiment, track history, and restore previous versions whenever necessary.

I also learned that GitHub is the central platform where ORION's code will be stored, managed, reviewed, and shared. It will become the single source of truth for the project and allow future collaboration as the product grows.

I installed Python and understood that it will be the primary programming language used to build ORION's backend, automation engine, integrations, and AI capabilities.

I installed Visual Studio Code, which will become my primary development environment for writing, testing, debugging, and maintaining ORION's source code.

Finally, I created the engineering workspace and GitHub profile for ORION SECURITY LABS LTD. For the first time, the project feels like a real software company rather than just an idea.

What Challenged Me

Today's biggest challenge was understanding how all the engineering tools fit together.

Coming from a SOC and Incident Response background, I am comfortable investigating security incidents, but software engineering is still new to me. At times, Git, GitHub, Python, VS Code, repositories, and source control felt like separate pieces that I couldn't fully connect.

However, as the day progressed, those pieces slowly started making sense. I now understand that every tool has a specific role in building ORION, and I know that with daily practice my confidence will continue to grow.

I also realised that learning software engineering is a journey, and I do not expect to master everything in a single day.

What Excites Me

The most exciting part of today's learning is realising that ORION is no longer just an idea—it is becoming a real product with a real company behind it.

In the next two to three years, I see myself becoming a well-rounded cybersecurity engineer who can confidently work as a Senior Incident Response Analyst, Detection Engineer, Security Engineer, AI Security Engineer, and eventually a Security Consultant.

More importantly, I want to become someone who can walk into any organisation, understand their security challenges, and confidently design practical solutions without fear.

I also want ORION to become a platform that genuinely helps SOC analysts investigate alerts faster, reduce manual effort, and make better security decisions.

Tomorrow's Goal

Tomorrow I want to create ORION's first Git repository, make my first commit, and begin writing my first Python program.

More importantly, I want to continue developing the engineering mindset that will eventually allow me to build ORION into a real security platform.
