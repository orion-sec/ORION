# ORION Engineering Journal – Day 5

## Goal

Extend the IOC Extractor to automatically identify multiple IOC types from investigation text.

---

## Features Added

Successfully implemented extraction for:

- URLs
- IPv4 Addresses
- MD5 Hashes
- SHA1 Hashes
- SHA256 Hashes
- Email Addresses
- Domains

---

## New Python Concepts Learned

### import re

Learned how to import Python's Regular Expression library.

---

### re.findall()

Used Regular Expressions to search investigation text for matching patterns.

Example:

```python
urls = re.findall(...)
```

---

### Enumerate

Learned to number IOC results automatically.

Example:

```python
for number, url in enumerate(urls, start=1):
```

---

### Regex

First practical experience using Regular Expressions.

Patterns created for:

- URL
- IPv4
- MD5
- SHA1
- SHA256
- Email
- Domain

---

## Test Result

Sample investigation text successfully produced:

- URL list
- IP list
- Hash list
- Email list
- Domain list

---

## Observation

Current Domain Regex incorrectly detects the username portion of email addresses (for example `john.smith`) as a domain.

This will be improved in a future version.

---

## Lessons Learned

Regex is extremely powerful for security automation.

Small amounts of Python can automatically extract indicators from large investigation notes, saving analysts significant manual effort.
