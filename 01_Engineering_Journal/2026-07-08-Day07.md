# ORION Engineering Journal

## Day 07 – Modular Software Engineering (Phase 2)

**Date:** 08 July 2026

---

# Objective

Continue transforming ORION IOC Extractor into a professional software project by separating responsibilities into reusable Python modules.

The focus today was improving software architecture rather than adding new IOC extraction capabilities.

---

# Work Completed

## 1. Created Modular Project Structure

Created a dedicated module directory:

11_Modules/

This directory now stores reusable Python modules instead of placing every function inside the main application.

---

## 2. Built display.py

Created a dedicated display module.

The following reusable functions were moved into this module:

- display_summary()
- display_results()
- display_all_results()
- display_report()

Purpose:

Separate all output formatting from the main application.

This means the main application no longer knows how reports are displayed.

Instead, it simply passes data to the display module.

---

## 3. Built extract.py

Created a reusable extraction module.

Function created:

extract_iocs()

Responsibilities:

- Extract URLs
- Extract IP Addresses
- Extract MD5 hashes
- Extract SHA1 hashes
- Extract SHA256 hashes
- Extract Email Addresses
- Extract Domains
- Remove duplicate values
- Return structured IOC results

---

## 4. Simplified the Main Application

The original IOC extractor contained extraction logic, display logic and application flow in one file.

After today's refactor the application now performs only four responsibilities:

1. Display application banner

2. Receive investigation text

3. Call extract_iocs()

4. Pass results to display_report()

This significantly reduced the amount of code inside the main script.

---

## 5. Learned Custom Module Imports

Imported reusable functions from custom modules.

Examples:

from extract import extract_iocs

from display import display_report

---

## 6. Learned Python Module Search Path

Added the custom modules directory using:

sys.path.append()

This allows Python to locate modules stored outside the current folder.

---

## 7. Improved Data Flow

Previous architecture:

Investigation

↓

Extract IOCs

↓

Return seven separate variables

↓

Main program builds dictionary

↓

Display results

New architecture:

Investigation

↓

extract_iocs()

↓

Return one dictionary

↓

display_report()

This approach is cleaner and much easier to maintain.
