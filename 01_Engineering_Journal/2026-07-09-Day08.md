# Objective

Today's objective was to begin Phase 2 of ORION by introducing the first enrichment capability. Instead of only extracting Indicators of Compromise (IOCs), ORION now enriches IP addresses with additional security context before presenting the investigation results.

---

# What I Built

Today I created the first enrichment module for ORION.

New module created:

11_Modules/enrich.py

This module enriches every extracted IP address using Python's built-in `ipaddress` library.

The enrichment module determines whether an IP address is:

- IPv4 or IPv6
- Public or Private
- Global
- Loopback
- Multicast
- Reserved

The enriched information is automatically added to the investigation results before the final report is displayed.

---

# Program Improvements

The IOC extraction workflow has now evolved from:

Investigation
→ Extract IOCs
→ Display Results

to

Investigation
→ Extract IOCs
→ Enrich IP Addresses
→ Generate Investigation Report

The main script now passes extracted IP addresses to the enrichment module before generating the final report.

# Challenges Encountered

During implementation I encountered several software engineering challenges:

- Duplicate function definitions after refactoring.
- Incorrect placement of function calls.
- Variable scope issue causing a NameError.
- Indentation errors.
- Understanding which function should own specific logic.
- Learning how modules communicate using dictionaries.

Each issue was resolved through debugging rather than rewriting the program.

---

# Lessons Learned

Today's biggest lesson was understanding program structure.

I learned that:

- Functions should perform one responsibility only.
- Reusable modules make future development easier.
- Dictionaries allow information to flow between modules.
- Function calls must exist inside the correct scope.
- Debugging is often about architecture rather than syntax.

I also became much more comfortable reading existing code and knowing where new functionality belongs.
