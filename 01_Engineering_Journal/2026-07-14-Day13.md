## Modules Added

### 1. URL Enrichment Module

Created:

`11_Modules/url_enrich.py`

Purpose:

- Parse extracted URLs into structured components.
- Identify the URL scheme.
- Extract the hostname.
- Extract the port.
- Extract the path.
- Extract the query string.

Example input:

`https://login.example.com:443/account/reset?user=sam`

Example enrichment:

- Scheme: `https`
- Hostname: `login.example.com`
- Port: `443`
- Path: `/account/reset`
- Query: `user=sam`

This provides structured URL context that can later support deeper security analysis and external threat-intelligence integrations.

---

### 2. URL Risk Scoring Module

Created:

`11_Modules/url_score.py`

Purpose:

- Analyse URL characteristics.
- Identify suspicious security-related keywords.
- Assign a preliminary risk score.
- Record the reasons contributing to the score.

The test URL contained:

- `login`
- `account`
- `reset`

ORION detected these characteristics and generated:

- Risk: `Low`
- Score: `15`

Reasons:

- Suspicious URL keyword detected: `login`
- Suspicious URL keyword detected: `account`
- Suspicious URL keyword detected: `reset`

Important design principle:

A suspicious keyword alone does not prove malicious activity. The URL scoring engine provides investigation context rather than making a final malicious verdict.

## Debugging Incident — Domain Intelligence Confidence Overwrite

During testing, ORION correctly detected:

`matched_keywords: ['login']`

However, the domain intelligence output incorrectly returned:

- Reputation: `Unknown`
- Confidence: `0`

The expected result was:

- Reputation: `Suspicious`
- Confidence: `40`

### Investigation Method

The issue was isolated by testing the domain intelligence module directly:

`inspect_domain('login.example.com')`

The isolated test confirmed that:

- Keyword detection was working.
- `login` was successfully added to `matched_keywords`.
- The final reputation and confidence values were still incorrect.

### Root Cause

The issue was caused by incorrect Python indentation.

The intended logic was:

```text
IF suspicious keywords are matched:
    Reputation = Suspicious
    Confidence = 40
ELSE:
    Reputation = Unknown
    Confidence = 0
