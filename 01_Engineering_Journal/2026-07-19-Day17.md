# Day 17 – Behavioural Evidence Analysis & Response Playbook Engine

## Objective

Today's objective was to enhance ORION's investigation engine by introducing Behavioural Evidence Analysis and implementing the first version of the Response Playbook Engine.

The goal was to move beyond IOC-only reasoning by allowing ORION to analyse investigation narrative and user behaviour, improving attack pattern detection while also generating structured response playbooks aligned with real-world incident response activities.

---

## Work Completed

### Behavioural Evidence Analysis

The Attack Pattern Detection module was enhanced to analyse investigation narrative rather than relying exclusively on IOC scores.

A new behavioural evidence engine was introduced which evaluates investigation text for indicators of credential compromise.

Current behavioural indicators include:

- entered credentials
- submitted credentials
- entered password
- submitted password
- provided credentials
- credential harvesting
- login details entered

This enables ORION to reason from analyst observations rather than only technical indicators.

---

### Credential Phishing Detection Enhancement

The Credential Phishing detection logic was redesigned.

Previous implementation required:

Medium/High URL Risk

AND

Suspicious Domain

This approach failed to detect phishing investigations containing low-risk URLs.

The new logic now evaluates:

Credential Submission Behaviour

AND

(Suspicious URL OR Suspicious Domain)

This significantly improves phishing detection accuracy while reducing dependence on URL reputation alone.

---

### Safe Threat Correlation Handling

A defensive programming improvement was implemented to support investigations without IP addresses.

Previously, investigations containing no IP indicators generated an IndexError because ORION attempted to access a Threat Correlation object that did not exist.

A default Threat Correlation object is now created whenever no IP-based intelligence is available.

Benefits include:

- Improved engine stability
- Support for URL-only investigations
- Support for phishing investigations
- Improved fault tolerance

---

### Response Playbook Engine

Created a new module:

11_Modules/response_playbooks.py

The Response Playbook Engine generates structured response actions based on detected attack patterns.

Implemented playbooks include:

Malware Delivery

- Identify affected endpoint
- Review EDR telemetry
- Isolate endpoint
- Block malicious URL
- Block malicious IP
- Block malicious hash
- Hunt for related indicators
- Collect forensic evidence
- Escalate to Incident Response

Credential Phishing

- Identify affected users
- Review email telemetry
- Remove phishing emails
- Block URL, domain and sender
- Review sign-in activity
- Revoke active sessions
- Reset passwords
- Review MFA registration
- Review inbox rules
- Review OAuth consent
- Hunt for related indicators
- Escalate to Incident Response

## Engineering Lessons

Today's work reinforced several important engineering principles.

Behavioural evidence is often more valuable than IOC reputation.

Detection engines should combine:

- Technical indicators
- Threat intelligence
- User behaviour
- Contextual evidence

rather than depending on any single source.

The implementation of defensive programming also demonstrated the importance of designing resilient software capable of processing incomplete investigations without failure.
