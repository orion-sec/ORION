# Work Completed

## 1. Contextual Risk Assessment

A new module named `context_risk.py` was created.

The module evaluates investigation context rather than relying solely on IOC severity.

Current evidence considered includes:

- Credential submission
- MFA changes
- Suspicious authentication
- Mailbox rules
- Sensitive mailbox access

Each observation contributes towards a contextual risk score.

Current output includes:

- Risk Score
- Severity
- Confidence
- Containment Recommendation
- Supporting Evidence

This allows ORION to explain why a risk level was assigned instead of simply returning a severity label.

## 5. Domain Extraction Improvement

Testing exposed an architectural issue.

Example investigation:

john.smith@contoso.com

Previous behaviour:

Domains Found

- john.smith
- contoso.com
- login.example.com

The local portion of the email address was incorrectly interpreted as a DNS domain.

The extraction process was redesigned.

Current process:

1. Extract email addresses.
2. Temporarily remove email addresses from the investigation text.
3. Search remaining text for domains.
4. Extract domains from email addresses separately.
5. Merge both domain collections.
6. Remove duplicates while preserving order.

Current output:

Domains Found

- login.example.com
- contoso.com

This better reflects how analysts distinguish between identity information and infrastructure indicators.
